# Quick Start Recipe

Status: Verified
Last Updated: 2026-03-02
Source: curated web research

## Goal

给第一次真正上手 OpenClaw 的人一个最短路径。

## Success Criteria

完成这篇后，你应该做到这几件事：

- 知道先看哪些官方页面
- 按正确顺序开始，而不是乱点功能
- 在接渠道或装插件前，先建立安全边界
- 明白第一阶段的目标不是“全都配好”，而是“跑通最小路径”

## The Short Path

1. Read the official overview
2. Choose the install path
3. Run the minimum setup
4. Learn the channel model
5. Apply the security baseline
6. Only then expand with skills, plugins, or workflows

## Step 1: Start from the official docs

- Official source: <https://docs.openclaw.ai/>
- Why first: 先建立全局心智模型，避免把 OpenClaw 当成单一聊天工具或插件列表。

## Step 2: Follow the official install path

- Official source: <https://docs.openclaw.ai/install>
- Why second: 安装方式和运行环境是后面所有步骤的基础，先按官方路径走，先别抄第三方教程。

## Step 3: Use the official getting started flow

- Official source: <https://docs.openclaw.ai/start/getting-started>
- Why third: 这一步的目标不是做完整系统，而是先跑通最小闭环，确认你能进入 OpenClaw 的基本运行面。

## Step 4: Learn channels as entry points

- Official source: <https://docs.openclaw.ai/channels/index>
- Why fourth: 很多人一开始就把渠道当主系统，但官方结构里渠道只是入口。先理解这一点，后面配置和排障会稳很多。

## Step 5: Apply the security baseline before expansion

- Official source: <https://docs.openclaw.ai/gateway/security>
- Why fifth: 在你接更多渠道、给更多权限、安装更多扩展之前，先把 access、scope 和 exposed surfaces 想清楚。

## What To Avoid In Week 1

- 一上来就接太多 channels
- 还没搞清权限边界就公开暴露入口
- 装很多 skills 或 plugins，但不清楚信任边界
- 还没跑通最小路径就开始调模型和 provider
