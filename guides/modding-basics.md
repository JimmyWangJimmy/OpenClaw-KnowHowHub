# Modding Basics

Status: Draft
Last Updated: 2026-03-01
Source: curated web research

## Goal

如果你的目标是进入 Captain Claw / OpenClaw 的自定义内容制作，这一页给出最小可行路径。

## What Counts As "Modding" Here

当前社区里的 “modding” 往往不是单一概念，而是几条不同路线：

- 自定义关卡制作
- 基于 CrazyHook 的扩展玩法
- 资源和视觉元素的定制
- 对原有关卡和机制的研究与再实现

## Recommended Starting Route

### 1. Stabilize Your Base Setup

- Install path: <https://captainclaw.net/en/downloads.html>
- Support path: <https://captainclaw.net/en/support.html>
- Why first: 如果基础运行环境不稳定，后续的自定义内容测试会非常痛苦。

### 2. Learn The Current Editing Workflow

- WapMap: <https://github.com/Zax37/WapMap>
- Why second: 当前主线制作工作流围绕 WapMap，而不是旧教程里常见的旧工具。

### 3. Read The Level Designer's Guide

- Primary source: <https://captainclaw.net/en/downloads.html>
- Why third: 这是目前最系统的实践资料之一，能让你把“改点东西”变成“理解社区工作流”。

### 4. Study Existing Custom Levels

- Directory: <https://captainclaw.net/en/clevels.html>
- Why fourth: 现成作品能告诉你社区已经接受什么样的结构、难度、依赖和扩展方式。

## Suggested First Project

最合理的第一个项目不是“大改原版”，而是：

1. 安装并确认基础游戏可稳定运行
2. 打开 WapMap
3. 研究一个简单的现有关卡
4. 做一个很小的修改或一个很短的原创测试地图

## Where CrazyHook Fits

- Primary sources:
  - <https://captainclaw.net/en/downloads.html>
  - <https://captainclaw.net/en/archive.html>
- Why it matters: 现代社区中很多高级内容、推荐关卡和扩展玩法都默认以 CrazyHook 为基础，因此它更像当前生态的基线，而不是附加组件。

## Common Mistakes

- 一上来就尝试做完整大型关卡
- 只看旧教程，不核对当前主流工具
- 不先确认运行环境稳定就开始做内容
- 不研究现有优秀关卡就直接开工
