## Goal

本页用于说明为什么 `Level Designer's Guide` 值得单独当作一个学习入口，以及读它之前和之后分别该做什么。

## Primary Source

### Downloads Page

- Link: <https://captainclaw.net/en/downloads.html>
- Why it matters: 下载页明确说明这是一个完整的八部分设计指南，由 Pejti 编写，覆盖从原版 WapWorld 到新 WapMap、CrazyHook 自定义逻辑和外部资产导入。

## Why This Guide Matters

很多资料只是告诉你“工具在哪”或“作品在哪”，但 `Level Designer's Guide` 更接近一条系统学习路径。它的价值在于：

- 把零散技巧组织成连续的设计学习路径
- 连接旧工具背景和当前主线工具
- 把 CrazyHook 和外部资产这类进阶内容纳入同一语境

## Best Time To Read It

### Read it after:

- [Tools Comparison](tools-comparison.md)
- [Level Editing](level-editing.md)

### Use it before:

- 直接尝试做大型原创关卡
- 盲目模仿旧教程里的过时工作流
- 在不了解 CrazyHook 基线的情况下设计复杂内容

## Suggested Reading Path

1. Understand the current tool baseline
2. Open the downloads page and locate the guide
3. Read it as a workflow guide, not just a tool manual
4. Compare what you read against existing custom levels
5. Use a small test map to validate what you learned
