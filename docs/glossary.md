# Glossary

Status: Verified
Last Updated: 2026-03-21
Source: repository planning

## OpenClaw

本仓库关注的核心对象。语境上指 OpenClaw 的 gateway-centered、agent-native 系统能力，而不是单一聊天应用。

## Guide

以完成某个任务为目标的操作型文档。

## Reference

以查阅为目的的结构化资料，不强调流程，强调准确和可定位。

## Research

用于保存实验过程、逆向记录、未完全确认的分析结果。

## Red Book

方法论层文档，回答“应该怎么想、怎么判断、先建立哪些边界”。

## Cookbook

配方式文档，回答“具体怎么做、按什么顺序做”。

## Playbook

可重复执行的任务流程定义，通常包含 trigger、inputs、steps、output contract、failure path。

## Skill

可安装的能力单元或扩展组件，必须按信任边界进行审查，不应等同于“安全可用”。

## Trust Boundary

系统中默认互信范围的边界定义。超出边界时应拆分系统或收紧权限，而不是默认共享高权限执行面。
